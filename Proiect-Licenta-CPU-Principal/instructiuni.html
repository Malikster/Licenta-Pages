Configurare completa pentru CPU Principal

Aceasta pagina aduna intr-un singur flux pasii de instalare pentru sistemul Linux de pe Raspberry Pi, modul Access Point, brokerul MQTT, serviciul de orchestrare si aplicatia pentru ecranul tactil.

Inapoi la prezentare Linux + ecran Mod AP Broker MQTT Servicii Verificare
Presupuneri: Raspberry Pi 4, ecran rezistiv 3.5 inch K-0403/52Pi, Raspberry Pi OS 64-bit oficial, retea locala gestionata de Pi si broker MQTT local.

1. Instalare Linux si suport pentru ecran

  1. 1

    Pregateste cardul microSD

    Foloseste Raspberry Pi OS (64-bit) din sursa oficiala Raspberry Pi.

    Choose Device -> Raspberry Pi 4
Choose OS -> Raspberry Pi OS (64-bit)

    Linkuri utile: Raspberry Pi Documentation - Installing images | Raspberry Pi Software / Imager

  2. 2

    Scrie imaginea pe card

    Foloseste Raspberry Pi Imager sau balenaEtcher. Recomandarea principala este Raspberry Pi Imager.

    Pentru imaginea folosita in proiect: hostname rpi-licenta, utilizator rpi-licenta, parola licenta.

  3. 3

    Primul boot

    Monteaza ecranul pe GPIO si porneste sistemul. Pe Raspberry Pi OS oficial este normal sa folosesti initial HDMI sau o configurare asistata, deoarece driverul LCD nu este inca instalat.

    Credentiale initiale pentru aceasta configuratie: utilizator rpi-licenta, parola licenta.

    whoami
hostname
pwd
sudo -v

    Test rapid dupa login: whoami si hostname trebuie sa afiseze rpi-licenta, iar sudo -v trebuie sa accepte parola licenta.

  4. 4

    Actualizare si configurare initiala

    sudo apt update
sudo apt full-upgrade -y
sudo raspi-config

    Configureaza localizarea, timezone-ul, tastatura si tara Wi-Fi. Parola poate fi schimbata aici ulterior, daca vrei.

  5. 5

    Instalare automata driver LCD

    cd ~
git clone https://github.com/Malikster/Proiect-Licenta-CPU-Principal.git
cd Proiect-Licenta-CPU-Principal
sudo bash 52pi/install.sh

    Varianta recomandata in repo: deschide meniul central de instalare. Pentru display, alege 1. Instalare ecran + touch.

  6. 6

    Instalare manuala driver LCD

    sudo rm -rf LCD-show
git clone https://github.com/goodtft/LCD-show.git
chmod -R 755 LCD-show
cd LCD-show
sudo ./LCD35-show

    Aceasta pastraza sistemul oficial Raspberry Pi OS, dar adauga manual driverul third-party pentru display.

  7. 7

    Calibrare touchscreen

    sudo apt install -y xinput-calibrator

    Daca ai folosit scriptul 52pi/install.sh, pachetul este deja instalat.

2. Configurare Raspberry Pi in modul Access Point

Varianta rapida: cloneaza repo-ul, ruleaza sudo bash 52pi/install.sh si alege 2. Configurare Raspberry Pi in modul Access Point.

Configuratia recomandata pentru proiect:

SSIDLicenta-RPi-AP
Parolalicenta-rpi
Interfatawlan0
IP AP192.168.4.1/24
nmcli device status
sudo nmcli connection add type wifi ifname wlan0 con-name Licenta-RPi-AP autoconnect yes ssid Licenta-RPi-AP
sudo nmcli connection modify Licenta-RPi-AP 802-11-wireless.mode ap 802-11-wireless.band bg ipv4.method shared ipv6.method disabled
sudo nmcli connection modify Licenta-RPi-AP wifi-sec.key-mgmt wpa-psk
sudo nmcli connection modify Licenta-RPi-AP wifi-sec.psk "licenta-rpi"
sudo nmcli connection modify Licenta-RPi-AP ipv4.addresses 192.168.4.1/24
sudo nmcli connection up Licenta-RPi-AP
Dupa configurare, Raspberry Pi devine nodul central de retea pentru restul subsistemelor. Pentru WPA-PSK, parola trebuie sa aiba minimum 8 caractere.

3. Instalare broker MQTT

Varianta rapida: cloneaza repo-ul, ruleaza sudo bash 52pi/install.sh si alege 3. Instalare broker MQTT.

Instalare Mosquitto

sudo apt install -y mosquitto mosquitto-clients
sudo systemctl enable mosquitto
sudo systemctl start mosquitto

Securizare minima recomandata

sudo mosquitto_passwd -c /etc/mosquitto/passwd licenta
sudo nano /etc/mosquitto/conf.d/licenta.conf
per_listener_settings false
allow_anonymous false
password_file /etc/mosquitto/passwd

listener 1883
protocol mqtt

listener 9001
protocol websockets

Listenerul 1883 este pentru MQTT clasic, iar 9001 este pentru MQTT over WebSocket folosit de site. Ambele folosesc acelasi fisier de parole.

Test rapid

mosquitto_sub -h localhost -t test/local
mosquitto_pub -h localhost -t test/local -m "Salut de pe Raspberry Pi"
mosquitto_sub -h localhost -u licenta -P licenta -t test/local
mosquitto_pub -h localhost -u licenta -P licenta -t test/local -m "Salut de pe Raspberry Pi"

Test pentru topicul de sonerie, util de pe laptop cand esti conectat la reteaua AP:

mosquitto_sub -h 192.168.4.1 -p 1883 -u licenta -P licenta -t 'LICENTA/SONERIE/COMANDA' -v

Cand apesi butonul Trimite soneria din tabul Sonerie, trebuie sa apara un JSON cu event, source si timestamp.

Daca Mosquitto nu porneste si vezi status=13, corecteaza permisiunile pentru /etc/mosquitto/passwd: 
sudo chown root:mosquitto /etc/mosquitto/passwd
sudo chmod 640 /etc/mosquitto/passwd
sudo systemctl restart mosquitto

4. Instalare servicii din proiect

Varianta rapida: cloneaza repo-ul, ruleaza sudo bash 52pi/install.sh si alege 4. Instalare servicii din proiect.

Serviciul de orchestrare MQTT

Instaleaza serviciul din scripts/:

git clone https://github.com/Malikster/Proiect-Licenta-CPU-Principal.git
cd Proiect-Licenta-CPU-Principal
sudo bash scripts/install_mqtt_status_service.sh
sudo nano /etc/default/mqtt-status-service
sudo systemctl restart mqtt-status-service.service

Acest serviciu monitorizeaza topicurile MQTT, mentine stare interna si republica statusurile relevante. Topicuri importante: LICENTA/ILUMINAT/STATUS, LICENTA/ACCES/STATUS (JSON retained) si LICENTA/SANITAR/STATUS. Pentru sonerie, aplicatia de ecran publica evenimente pe LICENTA/SONERIE/COMANDA.

Aplicatia pentru ecran

Instaleaza aplicatia locala din ecran/:

sudo bash ecran/install_ecran_app.sh
sudo nano /etc/default/ecran-app
sudo systemctl restart ecran-app.service

UI-ul local ruleaza pe display-ul Raspberry Pi, cu tab-uri pentru Iluminat, Acces, Sanitar, Sonerie si Setari. Butonul Wi-Fi din Setari poate crea si porni profilul AP daca lipseste. La boot, serviciul asteapta sesiunea grafica locala si fisierul .Xauthority inainte sa porneasca aplicatia. Installerul pune si o iconita pe desktop, plus autostart prin launcherul sesiunii grafice.

5. Configuratii importante

Poti precompleta rapid fisierele de configurare din meniul scriptului prin optiunea 5. Configuratii importante.

Configuratie MQTT service

MQTT_BROKER_HOST=127.0.0.1
MQTT_BROKER_PORT=1883
MQTT_USERNAME=licenta
MQTT_PASSWORD=licenta
SANITAR_SIMULATION_ENABLED=false
SANITAR_SIM_TANK_CAPACITY_L=10
SANITAR_SIM_INITIAL_R1_L=10
SANITAR_SIM_INITIAL_R2_L=0
SANITAR_SIM_FLOW_L_PER_MIN=1

Configuratie aplicatie ecran

MQTT_BROKER_HOST=127.0.0.1
MQTT_BROKER_PORT=1883
MQTT_USERNAME=licenta
MQTT_PASSWORD=licenta
ACCES_CAMERA_URL=http://192.168.4.3/camera.jpg
WIFI_AP_CONNECTION=Licenta-RPi-AP
WIFI_AP_SSID=Licenta-RPi-AP
WIFI_AP_PASSWORD=licenta-rpi
WIFI_AP_ADDRESS=192.168.4.1/24
WIFI_CLIENT_CONNECTION=netplan-wlan0-MLK
WIFI_INTERFACE=wlan0

Daca actualizezi fisiere din ecran/, reruleaza sudo bash ecran/install_ecran_app.sh, nu doar restartul serviciului, pentru ca installerul actualizeaza si helperul Wi-Fi si regula din /etc/sudoers.d/.

Pentru simularea sanitara, meniul principal are acum optiunea 9. Configurare simulare sanitara, care modifica valorile SANITAR_SIM_* din /etc/default/mqtt-status-service si reporneste serviciul.

6. Integrarea cu Acces-Supraveghere

Sub-proiectul Acces-Supraveghere expune:

  • /camera.jpg pentru preview-ul camerei
  • /status pentru status HTTP
  • /garage/open, /garage/stop, /garage/close
  • /gate/unlock, /gate/lock

In exemplul de configurare statica din acel sub-proiect, placa foloseste IP-ul 192.168.4.3, iar UI-ul din ecran/ foloseste implicit http://192.168.4.3/camera.jpg.

7. Verificare finala

Varianta rapida: cloneaza repo-ul, ruleaza sudo bash 52pi/install.sh si alege 6. Verificare status servicii.

Servicii de sistem

sudo systemctl status mosquitto
sudo systemctl status mqtt-status-service.service
sudo systemctl status ecran-app.service
sudo ss -ltnp | grep -E '1883|9001'

Loguri live

sudo journalctl -u mosquitto -f
sudo journalctl -u mqtt-status-service.service -f
sudo journalctl -u ecran-app.service -f
Daca totul este configurat corect, Raspberry Pi va porni cu AP activ, broker MQTT functional, serviciul de orchestrare pornit si aplicatia locala afisata direct pe ecran.
