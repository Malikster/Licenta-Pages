Sistem automatizare instalatii electrice tip Smart-House folosind Arduino

Proiect Licenta - Instructiuni instalare si configurare

Ghid pentru programarea si configurarea modulului ESP12F_Relay_30A_X8 folosit in proiectul de iluminat.

Atentie: nu conecta sarcini de 230V la relee in timpul programarii. Fa programarea cu placa alimentata doar pentru flash si teste de comunicatie.

1. Ce iti trebuie

Hardware

  • 1 x ESP12F_Relay_30A_X8
  • 1 x Arduino UNO folosit ca adaptor USB-Serial
  • Cablu USB pentru Arduino UNO
  • Fire jumper mama-tata / tata-tata, in functie de conectori
  • Sursa pentru placa de relee, daca este necesara de placa folosita

Software

  • Arduino IDE
  • Pachetul de placi ESP8266 pentru Arduino IDE
  • Biblioteca PubSubClient by Nick O'Leary

2. Fisierele din proiect

  • Proiect_Licenta_Iluminat/Proiect_Licenta_Iluminat.ino - logica principala
  • Proiect_Licenta_Iluminat/config.h - configurarea WiFi, MQTT, topicuri, pini
  • README.md - rezumatul firmware-ului

3. Cum folosesti Arduino UNO ca adaptor USB-Serial

Arduino UNO nu va rula sketch-ul. El va fi folosit doar ca punte intre PC si ESP8266.

  1. Conecteaza Arduino UNO la PC prin USB.
  2. Dezactiveaza microcontrollerul UNO prin legarea pinului RESET la GND.
  3. Astfel, convertorul USB-Serial de pe UNO ramane disponibil, iar ATmega328P nu mai interfereaza cu liniile seriale.

4. Conexiunea intre PC -> Arduino UNO -> ESP12F_Relay_30A_X8

Arduino UNO ESP12F_Relay_30A_X8 Rol
GND GND Masa comuna
D0 / RX TX Date trimise de ESP catre PC
D1 / TX RX Date trimise de PC catre ESP
5V 5V Alimentare pentru modul, doar daca placa accepta 5V pe pinul respectiv
GND GPIO0 Activeaza modul de flash in timpul incarcarii
RESET legat la GND - Dezactiveaza microcontrollerul Arduino UNO
Observatie: la aceasta placa, GPIO0 trebuie dus la GND doar pentru programare. Dupa upload, scoate aceasta legatura si reporneste placa.

5. Ordinea recomandata pentru conectare

  1. Conecteaza UNO la PC prin USB.
  2. Leaga RESET de pe UNO la GND.
  3. Realizeaza conexiunile seriale UNO -> placa ESP.
  4. Leaga GPIO0 la GND pe placa ESP.
  5. Alimenteaza placa ESP.
  6. Deschide Arduino IDE si pregateste upload-ul.

6. Configurarea Arduino IDE

6.1 Instalare suport ESP8266

  1. Deschide File -> Preferences.
  2. In campul Additional Boards Manager URLs adauga: 
    http://arduino.esp8266.com/stable/package_esp8266com_index.json
  3. Apasa OK.
  4. Deschide Tools -> Board -> Boards Manager.
  5. Cauta ESP8266 si instaleaza pachetul.

6.2 Instalare biblioteca MQTT

  1. Deschide Sketch -> Include Library -> Manage Libraries.
  2. Cauta PubSubClient.
  3. Instaleaza biblioteca PubSubClient by Nick O'Leary.

6.3 Selectia placii si a portului

  • Tools -> Board -> ESP8266 Boards -> Generic ESP8266 Module
  • Tools -> Port -> selecteaza portul pe care apare Arduino UNO

7. Setari recomandate in Arduino IDE pentru upload

8. Configurarea firmware-ului

Deschide fisierul Proiect_Licenta_Iluminat/config.h si completeaza: 

const char* WIFI_SSID = "Licenta-RPi-AP";
const char* WIFI_PASSWORD = "licenta-rpi";
const IPAddress WIFI_LOCAL_IP(192, 168, 4, 2);
const IPAddress WIFI_GATEWAY(192, 168, 4, 1);
const IPAddress WIFI_SUBNET(255, 255, 255, 0);
const IPAddress WIFI_DNS(192, 168, 4, 1);
const char* MQTT_BROKER = "192.168.4.1";
const uint16_t MQTT_PORT = 1883;
const char* MQTT_USER = "licenta";
const char* MQTT_PASSWORD = "licenta";

Adresa IP fixa a modulului ESP in reteaua WiFi este: 

192.168.4.2

Topicurile folosite de firmware sunt: 

LICENTA/ILUMINAT/COMANDA
LICENTA/ILUMINAT/STATUS

9. Scrierea sketch-ului pe placa

  1. Deschide Proiect_Licenta_Iluminat/Proiect_Licenta_Iluminat.ino in Arduino IDE.
  2. Verifica faptul ca config.h este completat.
  3. Asigura-te ca GPIO0 este legat la GND.
  4. Apasa Verify pentru compilare.
  5. Apasa Upload.
  6. Daca este nevoie, reporneste placa exact in momentul inceperii upload-ului.
  7. Asteapta mesajul Done uploading.

10. Dupa upload

  1. Deconecteaza GPIO0 de la GND.
  2. Reporneste placa ESP.
  3. Deschide Serial Monitor la 115200 baud.
  4. Verifica mesajele de conectare la WiFi si brokerul MQTT.

11. Comportamentul firmware-ului

  • ESP se conecteaza la WiFi-ul creat de Raspberry Pi.
  • ESP se conecteaza la brokerul MQTT de pe Raspberry Pi.
  • Asculta topicul LICENTA/ILUMINAT/COMANDA.
  • Accepta comenzi de forma RELEU-X-Y.
  • La fiecare reconectare MQTT, asculta topicul LICENTA/ILUMINAT/STATUS, aplica primul mesaj valid de forma ABCDEFGH, apoi se dezaboneaza de la status pana la urmatoarea reconectare.

12. Exemple de mesaje MQTT

Comenzi

RELEU-1-1
RELEU-2-0
RELEU-8-1

Status publicat de Raspberry Pi

10000001

Interpretare: releul 1 pornit, releele 2-7 oprite, releul 8 pornit.

13. Probleme frecvente

  • Daca nu porneste upload-ul, verifica GPIO0 -> GND si reporneste placa.
  • Daca apar erori seriale, verifica inversarea corecta a pinilor TX/RX.
  • Daca placa nu intra pe WiFi, verifica valorile din config.h.
  • Daca releele nu revin la starea dorita dupa restart, Raspberry Pi trebuie sa retrimita sau sa retina mesajul de status.
  • Daca placa se reseteaza in timpul WiFi, verifica alimentarea. ESP8266 este sensibil la surse slabe.
