Sistem automatizare instalatii electrice tip Smart-House folosind Arduino

Firmware Instalatia Sanitara

Documentatie de folosire pentru firmware-ul ESP32 care gestioneaza doua rezervoare, senzori de nivel, debitmetru, valva, pompa bidirectionala, afisaj OLED si integrare MQTT.

Resurse asociate: prezentare proiect, schema MQTT si AUTO/MANUAL, schema hardware si alimentari, schema vizuala

ESP32 + OLED JSN-SR04T MQTT Auto / Manual

Scopul proiectului

Proiectul controleaza o instalatie sanitara experimentala cu doua rezervoare, denumite R1 si R2.

  • Linia 1: R1 -> debitmetru -> valva -> R2
  • Linia 2: R1 <-> pompa <-> R2

Firmware-ul masoara nivelurile, citeste debitul, comanda actuatorii, afiseaza local starea pe OLED si comunica prin MQTT.

Componente hardware necesare

Control

  • ESP32 Development Board cu OLED 0.96 inch
  • cablu USB pentru programare

Senzori

  • 2 x JSN-SR04T
  • 2 x plutitor
  • 1 x debitmetru

Actuatori

  • 1 x valva electromagnetica 12V NC
  • 1 x pompa bidirectionala
  • driver valva
  • driver pompa cu inversare sens

Alimentare

  • alimentare ESP32
  • alimentare separata pentru valva si pompa
  • masa comuna

Observatii pentru JSN-SR04T

  • ECHO trebuie coborat la 3.3V pentru ESP32 daca modulul este alimentat la 5V.
  • Exemplu divizor: 1k intre ECHO si ESP32, 2k intre ESP32 si GND.
  • Senzorul trebuie montat vertical si ferit de zone moarte aproape de nivelul maxim.

Configurare config.h

Fisierul contine setarile de retea, MQTT, timpii de refresh, filtrarea pentru JSN-SR04T, calibrarea debitmetrului si pragurile pentru modul AUTO.

Valori recomandate de retea

  • WiFi SSID: Licenta-RPi-AP
  • WiFi password: licenta-rpi
  • MQTT host: 192.168.4.1
  • MQTT port: 1883
  • client ID: licenta-sanitar-esp32
constexpr char WIFI_SSID[] = "Licenta-RPi-AP";
constexpr char WIFI_PASSWORD[] = "licenta-rpi";
constexpr char MQTT_HOST[] = "192.168.4.1";
constexpr uint16_t MQTT_PORT = 1883;
constexpr char MQTT_CLIENT_ID[] = "licenta-sanitar-esp32";

Biblioteci Arduino necesare

  • PubSubClient
  • Adafruit GFX Library
  • Adafruit SSD1306

Este necesar si suportul de placi ESP32 in Arduino IDE.

Compilare si upload in Arduino IDE

  1. Deschide Proiect-Licenta-Instalatia-Sanitara.ino in Arduino IDE.
  2. Instaleaza bibliotecile necesare.
  3. Selecteaza board-ul ESP32 Dev Module sau modelul real al placii.
  4. Selecteaza portul serial corect.
  5. Actualizeaza config.h daca este necesar.
  6. Apasa Verify.
  7. Apasa Upload.
  8. Deschide Serial Monitor la 115200 baud.

Topicuri MQTT

Comenzi

LICENTA/SANITAR/COMANDA

Status

LICENTA/SANITAR/STATUS

Mesaje acceptate

  • AUTO-1
  • AUTO-0
  • POMPA-A
  • POMPA-B
  • POMPA-0
  • VALVA-1
  • VALVA-0

Mesaje publicate

  • ABC pentru modul, valva si pompa
  • NIVEL-1-XYZ
  • NIVEL-2-XYZ
  • DEBIT-XYZ, daca debitul este disponibil in firmware

DEBIT-XYZ foloseste debitul real in L/min, rotunjit si formatat pe 3 cifre, de exemplu DEBIT-000 sau DEBIT-001.

Mod AUTO si MANUAL

AUTO

  • R1 mare -> valva deschisa
  • R1 mic -> valva inchisa
  • R2 mare -> pompa R2 -> R1
  • R2 mic -> pompa oprita

MANUAL

  • actuatorii sunt controlati din MQTT
  • la intrarea in MANUAL, valva se inchide
  • la intrarea in MANUAL, pompa se opreste

Troubleshooting

MQTT

  • verifica SSID, parola si brokerul 192.168.4.1:1883
  • verifica topicurile exacte
  • foloseste Serial Monitor la 115200

JSN-SR04T

  • protejeaza ECHO catre 3.3V
  • monteaza vertical senzorul
  • ajusteaza pragurile de distanta si filtrarea

OLED

  • verifica bibliotecile Adafruit
  • verifica pinii OLED_SDA, OLED_SCL, OLED_RST
  • verifica maparea interna a placii cu OLED integrat

Pompa

  • driverul trebuie sa permita ambele sensuri
  • conventia trebuie pastrata: POMPA-A = R1 -> R2, POMPA-B = R2 -> R1
  • daca directiile sunt inversate fizic, corecteaza cablarea sau driverul
