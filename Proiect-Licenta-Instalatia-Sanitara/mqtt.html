Comenzi acceptate

Mesaj Efect
AUTO-1Comuta firmware-ul in modul AUTO.
AUTO-0Comuta firmware-ul in modul MANUAL, inchide valva si opreste pompa.
POMPA-APorneste pompa in directia R1 -> R2.
POMPA-BPorneste pompa in directia R2 -> R1.
POMPA-0Opreste pompa.
VALVA-1Deschide valva.
VALVA-0Inchide valva.

Status publicat

Mesaj Semnificatie
ABC A = mod (0 manual, 1 auto), B = valva (0 inchisa, 1 deschisa), C = pompa (0 stop, 1 R1 -> R2, 2 R2 -> R1)
NIVEL-1-XYZNivelul rezervorului R1 in procente, pe 3 cifre.
NIVEL-2-XYZNivelul rezervorului R2 in procente, pe 3 cifre.
DEBIT-XYZDebit real in L/min, rotunjit si formatat pe 3 cifre.

Fluxul modurilor

1. Activare AUTO

Mesajul AUTO-1 trece firmware-ul in modul automat. Statusul este republicat imediat.

2. Decizie automata dupa nivel
  • daca nivelul in R1 depaseste pragul maxim, valva se deschide;
  • daca nivelul in R1 scade sub pragul minim, valva se inchide;
  • daca nivelul in R2 depaseste pragul maxim, pompa merge R2 -> R1;
  • daca nivelul in R2 scade sub pragul minim, pompa se opreste.
3. Activare MANUAL

Mesajul AUTO-0 trece firmware-ul in modul manual, inchide valva si opreste pompa imediat.

4. Control manual

In modul manual, comenzile POMPA-A, POMPA-B, POMPA-0, VALVA-1 si VALVA-0 sunt executate direct.

Conventii importante

  • POMPA-A trebuie sa ramana conventia pentru R1 -> R2.
  • POMPA-B trebuie sa ramana conventia pentru R2 -> R1.
  • Cand simularea din CPU Principal este oprita, valorile reale publicate de firmware devin sursa pentru afisaje.
  • Statusul se publica periodic si imediat dupa schimbari de pompa sau valva.
