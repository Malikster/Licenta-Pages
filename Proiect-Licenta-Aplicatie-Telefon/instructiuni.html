Aplicatia Android Control Casa pentru Acces, Iluminat si Instalatia Sanitara.

Scop Cerinte Build debug Instalare Configurare MQTT Pagini Troubleshooting

Scopul aplicatiei

Aplicatia Android este interfata mobila pentru controlul si monitorizarea subsistemelor proiectului de licenta.

  • Acces: garaj, poarta si camera de supraveghere.
  • Iluminat: controlul celor 8 circuite/relee.
  • Instalatia Sanitara: niveluri, valva, pompa, debit si mod AUTO/MANUAL.

Aplicatia comunica in principal prin MQTT cu brokerul local de pe Raspberry Pi si foloseste HTTP pentru imaginea camerei si fallback-ul de status al subsistemului Acces.

Cerinte

  • Android Studio instalat.
  • JDK compatibil cu proiectul Android.
  • Android SDK Platform 34 sau compatibil cu proiectul.
  • Telefon Android sau emulator.
  • Telefonul conectat la reteaua Raspberry Pi, de obicei Licenta-RPi-AP.
  • Broker MQTT disponibil in reteaua locala.

Build debug

Din radacina proiectului Proiect-Licenta-Aplicatie-Telefon, ruleaza:

.\gradlew.bat assembleDebug

APK-ul debug se genereaza in:

app/build/outputs/apk/debug/app-debug.apk

Instalare APK pe telefon

Prin Android Studio

  1. Conecteaza telefonul prin USB.
  2. Activeaza Developer options si USB debugging pe telefon.
  3. Deschide proiectul in Android Studio.
  4. Alege dispozitivul din bara de rulare.
  5. Ruleaza configuratia app.

Prin ADB

.\gradlew.bat assembleDebug
adb install -r app\build\outputs\apk\debug\app-debug.apk

Prin copiere APK

  1. Copiaza APK-ul pe telefon.
  2. Deschide APK-ul din managerul de fisiere.
  3. Permite instalarea din surse necunoscute daca Android solicita acest lucru.

Configurare initiala

La prima pornire, deschide setarile aplicatiei folosind butonul cu roata dintata din bara de sus.

Setare Valoare recomandata
Host MQTT tcp://192.168.4.1:1883
User MQTT licenta
Parola MQTT licenta
URL Acces-Supraveghere http://192.168.4.3
Refresh camera 2000 ms sau valoarea dorita

Setarea Debug este optionala. Cand este activata, aplicatia afiseaza informatii MQTT suplimentare si permite testarea comenzilor Acces fara autentificare biometrica.

Topicuri MQTT folosite

Topic Directie Rol
LICENTA/ACCES/GARAJ publish Comenzi garaj: inchide, stop, deschide
LICENTA/ACCES/POARTA publish Comenzi poarta: blocheaza, deblocheaza
LICENTA/ACCES/STATUS subscribe Status Acces in format JSON
LICENTA/ILUMINAT/COMANDA publish Comenzi relee iluminat
LICENTA/ILUMINAT/STATUS subscribe Status 8 biti pentru circuitele de iluminat
LICENTA/SANITAR/COMANDA publish Comenzi AUTO/MANUAL, valva si pompa
LICENTA/SANITAR/STATUS subscribe Status sanitar scurt, niveluri si debit

Format LICENTA/ACCES/STATUS

Payload-ul este JSON compatibil cu endpoint-urile /status si /api/access_status.

  • garageMoving
  • garageState
  • garagePositionSteps
  • garageTargetSteps
  • gateLocked
  • camera
  • mqtt
  • lastGarageCommand, optional
  • lastGateCommand, optional
  • source, optional
{
  "garageMoving": false,
  "garageState": "closed",
  "garagePositionSteps": 0,
  "garageTargetSteps": 0,
  "gateLocked": true,
  "camera": true,
  "mqtt": true,
  "lastGarageCommand": "inchide",
  "lastGateCommand": "blocheaza",
  "source": "CPUPrincipal"
}

Format LICENTA/ILUMINAT/STATUS

Payload-ul este un sir de 8 biti, de exemplu:

01010101

Format LICENTA/SANITAR/STATUS

Aplicatia accepta status scurt ABC, niveluri si debit.

110
NIVEL-1-075
NIVEL-2-040
DEBIT-001

DEBIT-000 se afiseaza ca 0 L/min, iar DEBIT-001 ca 1 L/min.

Pagini si utilizare

Acces

Controleaza garajul, poarta si afiseaza camera. Comenzile garajului sunt Inchide, Stop si Deschide. Poarta se controleaza cu Blocheaza / Deblocheaza.

Statusul Acces este actualizat din LICENTA/ACCES/STATUS, iar camera foloseste endpoint-ul /camera.jpg de la URL-ul configurat.

Iluminat

Pagina contine 8 butoane, cate unul pentru fiecare circuit. Comenzile sunt trimise prin MQTT, iar starea se actualizeaza din LICENTA/ILUMINAT/STATUS.

Pentru recuperarea starii dupa reconectare, CPU Principal ar trebui sa publice LICENTA/ILUMINAT/STATUS ca mesaj retained.

Instalatia Sanitara

Pagina afiseaza Nivel 1, Nivel 2, Valva, Pompa si Debit. Include comutator AUTO/MANUAL, control valva si control pompa: Pornire 1, Stop, Pornire 2.

Troubleshooting

Lipsa conexiunii MQTT

  1. Verifica daca telefonul este conectat la Licenta-RPi-AP.
  2. Verifica host-ul MQTT: tcp://192.168.4.1:1883.
  3. Verifica user/parola MQTT: licenta / licenta.
  4. Verifica daca brokerul MQTT ruleaza pe Raspberry Pi.
  5. Dezactiveaza temporar datele mobile daca telefonul evita reteaua locala.
  6. Apasa reconectare MQTT in aplicatie.

Camera inaccesibila

  1. Verifica daca telefonul este conectat la Licenta-RPi-AP.
  2. Verifica URL Acces-Supraveghere: http://192.168.4.3.
  3. Testeaza in browser: http://192.168.4.3/camera.jpg.
  4. Verifica daca subsistemul Acces-Supraveghere este pornit.

Telefon neconectat la Licenta-RPi-AP

  1. Deschide setarile Wi-Fi ale telefonului.
  2. Conecteaza-te la Licenta-RPi-AP.
  3. Daca Android avertizeaza ca reteaua nu are internet, alege sa ramai conectat.
  4. Dezactiveaza temporar datele mobile daca este necesar.
  5. Revino in aplicatie si apasa reconectare MQTT.
