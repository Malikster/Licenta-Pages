Instructiuni FireBeetle 2 ESP32-P4 + IO Expansion Board

Ghid dedicat variantei ESP-IDF pentru kitul FireBeetle 2 ESP32-P4, camera OV5640 pe MIPI-CSI, TF card, TMC2209 si micro servo.

Alege alta placa Schema hardware Arhitectura P4

1. Firmware recomandat

Pentru P4 foloseste subproiectul esp32-p4-ov5640-video-server, bazat pe ESP-IDF. Sketch-ul Arduino principal ramane doar pentru test rapid de actuatori pe P4, fara streaming camera MIPI.

  • stream OV5640 pe http://IP:81/stream
  • pagina de control acces pe http://IP/acces
  • status JSON pe http://IP/api/access_status
  • MQTT retained pe LICENTA/ACCES/STATUS

2. Alimentare

Sursa 12V
  +--> VMOT TMC2209
  +--> intrare buck 9-35V -> 5V

Buck 5V
  +--> 5V FireBeetle 2 ESP32-P4
  +--> 5V servo

GND comun:
  sursa 12V
  buck
  ESP32-P4
  TMC2209
  servo
Nu alimenta stepper-ul din 5V si nu alimenta servo-ul din 3.3V.

4. Pini rezervati pe P4

  • GPIO7/GPIO8: SDA/SCL, folositi si pentru camera/SCCB
  • GPIO39-45: TF card onboard
  • GPIO14-19, GPIO54, GPIO6: conexiunea interna cu modulul Wi-Fi/Bluetooth ESP32-C6
  • GPIO35: BOOT

5. Pregatirea TF card

  1. Formateaza cardul in FAT32.
  2. Copiaza optional esp32-p4-ov5640-video-server/sdcard/acces-p4.html in radacina cardului.
  3. Aplicatia monteaza cardul la /sdcard.

Daca fisierul /sdcard/acces-p4.html exista, pagina /acces il serveste pe acesta. Daca lipseste, firmware-ul foloseste pagina inclusa in binar.

6. ESP-IDF

In mediul local folosit in acest proiect, ESP-IDF este disponibil prin profilul Espressif instalat.

cd esp32-p4-ov5640-video-server
idf.py set-target esp32p4
idf.py menuconfig
idf.py build
idf.py -p PORT flash monitor

7. Endpoint-uri locale

  • http://IP/ pentru interfata video Espressif
  • http://IP/acces pentru pagina de control acces
  • http://IP:81/stream pentru stream MJPEG
  • http://IP/api/access_status pentru status JSON
  • http://IP/api/garage/open, /stop, /close
  • http://IP/api/gate/lock, /unlock

8. Test MQTT

mosquitto_pub -h 192.168.4.1 -u licenta -P licenta -t LICENTA/ACCES/GARAJ -m deschide
mosquitto_pub -h 192.168.4.1 -u licenta -P licenta -t LICENTA/ACCES/GARAJ -m stop
mosquitto_pub -h 192.168.4.1 -u licenta -P licenta -t LICENTA/ACCES/GARAJ -m inchide
mosquitto_pub -h 192.168.4.1 -u licenta -P licenta -t LICENTA/ACCES/POARTA -m deblocheaza
mosquitto_pub -h 192.168.4.1 -u licenta -P licenta -t LICENTA/ACCES/POARTA -m blocheaza
mosquitto_sub -h 192.168.4.1 -u licenta -P licenta -t LICENTA/ACCES/STATUS -v

LICENTA/ACCES/STATUS este publicat retained la conectare, dupa comenzi si la finalul miscarii garajului.

9. Troubleshooting

  • Daca stream-ul nu porneste, verifica flexul camerei OV5640 si configuratia MIPI-CSI.
  • Daca Wi-Fi/MQTT nu pornesc, verifica firmware-ul modulului intern ESP32-C6 si reteaua Raspberry Pi.
  • Daca stepper-ul vibreaza, verifica bobinele si curentul TMC2209.
  • Daca motorul merge invers, activeaza inversarea directiei din menuconfig.
  • Daca servo-ul reseteaza placa, imbunatateste alimentarea de 5V.
